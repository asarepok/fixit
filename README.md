# FixIt

A Ghana-focused artisan booking marketplace. Customers find verified plumbers,
electricians, mechanics and other artisans nearby, book them, pay by Mobile
Money through Paystack, and rate the job afterward. Built with Flutter,
Firebase, and Riverpod.

[Latest release](https://github.com/asarepok/fixit/releases/latest) has a
downloadable APK if you just want to try the app.

## Features

- **Search and booking** — location-aware artisan search, service categories, booking requests with status tracking
- **Roles** — customer, artisan, and admin accounts, each with their own dashboard and permissions
- **Artisan verification** — application flow with document upload, admin review and approval
- **Payments** — Mobile Money escrow via Paystack, released once a job is confirmed done
- **Chat** — a ticket-style thread per booking between customer and artisan
- **Reviews** — ratings with optional photos, shown on the artisan's profile alongside a work portfolio
- **Admin dashboard** — manage users, artisans, bookings, and payments in one place
- **Push notifications** — booking updates and messages delivered via FCM
- **Maps and location** — nearby-artisan search and a location picker backed by Google Maps and geocoding

## Architecture

Business logic lives away from the UI. Each screen only draws widgets and reads/calls a
provider. It never talks to Firebase, geolocation, or any other SDK directly.

```
lib/
├── main.dart                Firebase init + runApp only
├── firebase_options.dart    generated by FlutterFire CLI, don't hand-edit
│
├── app/                     app-wide wiring
│   ├── app.dart               root MaterialApp.router widget
│   ├── router.dart            GoRouter route table
│   ├── theme.dart              colors, text styles, ThemeData
│   ├── map_style.dart          Google Maps JSON style
│   └── constants.dart          AppConstants + route path constants
│
├── models/                  plain Dart domain types, no Firebase types allowed
│   ├── user_model.dart, booking_model.dart, chat_model.dart, payment_model.dart,
│   └── portfolio_model.dart, review_model.dart, verification_model.dart
│
├── services/                 the ONLY layer allowed to import firebase_auth,
│   ├── auth_service.dart        cloud_firestore, geolocator, etc. Thin wrappers around
│   ├── firestore_service.dart   the SDK, no business logic, no domain models in/out.
│   ├── storage_service.dart
│   ├── functions_service.dart   calls Cloud Functions (e.g. Paystack init/verify)
│   ├── notification_service.dart
│   └── location_service.dart
│
├── repositories/             business logic: combines services, returns domain models
│   ├── auth_repository.dart, artisan_repository.dart, booking_repository.dart,
│   ├── chat_repository.dart, payment_repository.dart, portfolio_repository.dart,
│   └── review_repository.dart, verification_repository.dart
│
├── providers/                Riverpod layer, the only thing screens read from/call
│   └── one per domain area, mirrors the repositories above, plus app_mode_provider,
│       location_provider, theme_provider
│
├── screens/                  UI only, no firebase_*/cloud_firestore imports, ever
│   ├── auth/          splash, onboarding, login, register, forgot_password
│   ├── customer/       home, search, bookings, booking details, profile, nearby artisans
│   ├── artisan/         dashboard, manage portfolio
│   ├── onboarding/      become-an-artisan application flow
│   ├── admin/            dashboard + manage users/artisans/bookings/payments
│   ├── chat/, maps/, settings/
│
├── widgets/                  shared widgets reused across screens
└── utils/                    helpers.dart (distance calc), validators.dart, extensions.dart

functions/                    Cloud Functions (TypeScript): Paystack init/verify/webhook,
                               notification triggers, anything that needs a secret key
```

**Rule:** if a file under `lib/screens/**` needs to import `firebase_auth`, `cloud_firestore`,
or `geolocator`, that logic belongs in a repository instead. Add a method there and call it
through a provider.

## Getting started

Prerequisites: Flutter SDK (see `environment.sdk` in `pubspec.yaml`), a Firebase project, and
a Paystack account for payments.

1. Install dependencies:
   ```
   flutter pub get
   ```
2. Connect your own Firebase project with the [FlutterFire CLI](https://firebase.google.com/docs/flutter/setup)
   (regenerates `lib/firebase_options.dart` and the platform config files):
   ```
   flutterfire configure
   ```
3. Deploy Firestore/Storage rules and Cloud Functions:
   ```
   firebase deploy --only firestore:rules,storage:rules,functions
   ```
4. Set your Paystack keys: the public key in `lib/app/constants.dart`, the secret key as a
   Cloud Functions config value (never committed, see the comment above `paystackPublicKey`).
5. Add a Google Maps API key in `android/app/src/main/AndroidManifest.xml` and
   `ios/Runner/AppDelegate.swift`.
6. Run the app:
   ```
   flutter run
   ```
