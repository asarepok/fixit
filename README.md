# FixIt GH

An artisan booking platform (verification, ratings, MoMo escrow) built with Flutter, Firebase, and Riverpod.

## Architecture

Business logic lives away from the UI. Each screen only draws widgets and reads/calls a
provider. It never talks to Firebase, geolocation, or any other SDK directly.

```
lib/
├── main.dart              Firebase init + runApp only
├── firebase_options.dart   generated by FlutterFire CLI, don't hand-edit
│
├── app/                    app-wide wiring
│   ├── app.dart              root MaterialApp.router widget
│   ├── router.dart           GoRouter route table
│   ├── theme.dart             colors, text styles, ThemeData
│   └── constants.dart         AppConstants + route path constants
│
├── models/                 plain Dart domain types, no Firebase types allowed
│   └── user_model.dart
│   (artisan_model.dart, booking_model.dart, message_model.dart, payment_model.dart,
│    review_model.dart: added when that feature is actually built)
│
├── services/                the ONLY layer allowed to import firebase_auth,
│   ├── auth_service.dart      cloud_firestore, or geolocator. Thin wrappers around the SDK,
│   ├── firestore_service.dart no business logic, no domain models in/out.
│   └── location_service.dart
│   (storage_service.dart, booking_service.dart, chat_service.dart, payment_service.dart,
│    notification_service.dart: added when that feature is built)
│
├── repositories/            business logic: combines services, returns domain models
│   ├── auth_repository.dart    register/login/logout + "my account" (role, profile, location)
│   └── artisan_repository.dart artisan search + nearby sorting
│   (booking_repository.dart, payment_repository.dart, review_repository.dart: added later)
│
├── providers/               Riverpod layer, the only thing screens read from/call
│   ├── auth_provider.dart, artisan_provider.dart, location_provider.dart, theme_provider.dart
│   (booking_provider.dart, chat_provider.dart: added later)
│
├── screens/                 UI only, no firebase_*/cloud_firestore imports, ever
│   ├── auth/        splash, onboarding, role_selection, login, register, forgot_password
│   ├── customer/     main_navigation, home, search, bookings, booking_details, profile,
│   │                 edit_profile, nearby_artisans, artisan_profile
│   ├── artisan/      dashboard (requests/jobs/profile/booking_details added when built)
│   ├── onboarding/   become_artisan (added when built)
│   ├── settings/, admin/, chat/, maps/
│
├── widgets/                shared widgets reused across screens
└── utils/                  helpers.dart (distance calc), validators.dart, extensions.dart
```

**Rule:** if a file under `lib/screens/**` needs to import `firebase_auth`, `cloud_firestore`,
or `geolocator`, that logic belongs in a repository instead. Add a method there and call it
through a provider.
